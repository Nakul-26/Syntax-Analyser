General C-like Syntax Grammar
=============================

Program structure
- program -> external_list
- external_list -> external external_list | epsilon
- external -> function_decl | function_def | statement

Functions
- function_decl -> type function_name ( parameter_list ) ;
- function_def -> type function_name ( parameter_list ) block
- function_name -> identifier | main
- parameter_list -> parameter (, parameter)* | epsilon
- parameter -> type identifier

Statements
- statement -> declaration
- statement -> assignment
- statement -> increment
- statement -> if_stmt
- statement -> while_stmt
- statement -> do_while_stmt
- statement -> for_stmt
- statement -> return_stmt
- statement -> break_stmt
- statement -> continue_stmt
- statement -> function_call_stmt
- statement -> block

Declarations
- declaration -> type decl_list ;
- type -> int | float | char
- decl_list -> decl_item (, decl_item)*
- decl_item -> identifier
- decl_item -> identifier = expr
- decl_item -> identifier [ number ]

Assignments and updates
- assignment -> lvalue = expr ;
- assignment_core -> lvalue = expr
- lvalue -> identifier
- lvalue -> identifier [ expr ]
- increment -> identifier ++ ;
- increment_core -> identifier ++

Blocks
- block -> { statement_list }
- statement_list -> statement statement_list | epsilon

Selection and iteration
- if_stmt -> if ( condition ) statement else_part
- else_part -> else statement | epsilon
- while_stmt -> while ( condition ) statement
- do_while_stmt -> do statement while ( condition ) ;
- for_stmt -> for ( for_init? ; condition? ; for_update? ) statement
- for_init -> declaration_core | assignment_core
- for_update -> increment_core | assignment_core

Conditions
- condition -> logical_or
- logical_or -> logical_and ( || logical_and )*
- logical_and -> relation ( && relation )*
- relation -> expr relop expr
- relation -> ( condition )
- relop -> > | < | >= | <= | == | !=

Expressions
- expr -> term (( + | - ) term)*
- term -> factor (( * | / ) factor)*
- factor -> identifier
- factor -> identifier [ expr ]
- factor -> function_call_expr
- factor -> number
- factor -> ( expr )

Function calls
- function_call_stmt -> function_call_expr ;
- function_call_expr -> function_name ( argument_list )
- argument_list -> expr (, expr)* | epsilon

Lexical notes
- identifier starts with a letter or underscore
- identifier may contain letters, digits, and underscores
- keywords are not accepted as identifiers
- number may be an integer literal or a floating-point literal

Context-aware validation
- break is valid only inside while, do-while, and for loops
- continue is valid only inside while, do-while, and for loops
- array access requires the target identifier to be declared as an array
- array indices must be integer expressions

AST notes
- declarations store type and identifier as child nodes
- declaration groups are used for comma-separated declarations
- assignments store target and expression as child nodes
- binary operators become expression nodes such as +, -, *, /
- conditions store the relational operator as the node value
- logical operators are stored as LOGICAL_OP nodes
